# Portafolio Web - Desarrollo Orientado a Servicios

## Descripción

Portafolio web profesional desarrollado como proyecto académico para la materia de **Aplicaciones Web a Servicios** del 5to semestre en la Universidad Tecnológica de Coahuila. 

Este proyecto presenta de manera estructurada y visual los conceptos fundamentales del desarrollo web orientado a servicios, incluyendo:
- Arquitectura SOA (Service Oriented Architecture)
- Implementación y consumo de APIs
- Desarrollo de APIs RESTful
- Despliegue y mantenimiento de aplicaciones web

El portafolio está diseñado con un enfoque profesional, utilizando una paleta de colores en tonos naranjas (#FF6B35) que refleja modernidad y profesionalismo, ideal para presentar en entrevistas laborales o como parte de un CV técnico.

Este proyecto demuestra habilidades en:
* Desarrollo Front-End moderno con Tailwind CSS
* Diseño responsive (mobile-first)
* Arquitectura de información clara y navegable
* Documentación técnica estructurada
* Diseño UI/UX orientado a portafolio profesional
* Deploy en GitHub Pages

## Tecnologías utilizadas

* **HTML5** - Estructura semántica del contenido
* **Tailwind CSS** - Framework de utilidades CSS para diseño responsive
* **JavaScript Vanilla** - Interactividad y navegación
* **Font Awesome 6** - Iconografía profesional
* **Google Fonts (Inter)** - Tipografía moderna y legible
* **GitHub Pages** - Hosting y despliegue gratuito

## Contenido de las Unidades

### Unidad 1: Introducción al Desarrollo Web Orientado a Servicios
- Conceptos fundamentales de SOA
- Servicios web y su importancia
- API, REST, JSON/XML
- Ventajas de la arquitectura orientada a servicios
- Ejemplos prácticos de aplicaciones e-commerce

### Unidad 2: Implementación de API
- Métodos HTTP (GET, POST, PUT, DELETE)
- Autenticación de APIs (API Keys, OAuth 2.0, JWT)
- Consumo de APIs externas
- Ejemplos de APIs populares (Google Maps, Stripe, Twitter, OpenWeather)
- Mejores prácticas de integración

### Unidad 3: Desarrollo de API
- Diseño de endpoints RESTful
- Códigos de estado HTTP
- Documentación de APIs (Swagger, Postman)
- Tecnologías backend (Node.js, Python, Java, PHP)
- Bases de datos relacionales y NoSQL

### Unidad 4: Implementación
- Plataformas de hosting (GitHub Pages, Netlify, Vercel, Heroku)
- CI/CD (Integración y Despliegue Continuo)
- Seguridad web (HTTPS, variables de entorno, validación)
- Optimización de rendimiento (CDN, caché, minificación)
- Monitoreo y mantenimiento de aplicaciones

## Pasos para clonar

1. Abrir la terminal o CMD
2. Ir a la carpeta donde quieras guardar el proyecto:

```bash
cd C:\Users\Usuario\Proyectos
```

3. Clonar el repositorio:

```bash
git clone https://github.com/wellington3100/MiPortafolio.git
```

4. Entrar a la carpeta del proyecto:

```bash
cd MiPortafolio
```

5. Abrir el archivo `index.html` en el navegador o visitar:

```
https://wellington3100.github.io/MiPortafolio
```

## Despliegue en GitHub Pages

### Pasos para activar GitHub Pages:

1. Ve a tu repositorio en GitHub: `https://github.com/wellington3100/MiPortafolio`
2. Click en **Settings** (Configuración)
3. En el menú lateral, busca **Pages**
4. En **Source**, selecciona **main** branch
5. Click en **Save**
6. Espera 1-2 minutos
7. Tu sitio estará disponible en: `https://wellington3100.github.io/MiPortafolio`

## Personalización

Para personalizar el portafolio:

1. **Cambiar colores**: Edita los valores en `tailwind.config` dentro de cada HTML
2. **Modificar contenido**: Actualiza el texto en cada sección
3. **Agregar proyectos**: Crea nuevas páginas siguiendo la estructura existente
4. **Cambiar tipografía**: Modifica el import de Google Fonts
